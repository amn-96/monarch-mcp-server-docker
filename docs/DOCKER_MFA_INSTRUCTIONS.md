# Monarch MCP Server Docker MFA Authentication Guide

The Monarch Money authentication process requires interactive input for Multi-Factor Authentication (MFA) codes. Additionally, the resulting session token is stored securely via the Python `keyring` library, which typically relies on OS-level secret managers (like macOS Keychain or Windows Credential Locker). 

Because Docker containers are headless and lack these OS-level native secret managers by default, we cannot easily pass a static `.env` file to replicate this process. Instead, we use `keyrings.alt` to store the token in a file inside the container, and we map this file to a persistent Docker volume so the server can reuse the token across restarts.

Here is how to complete the MFA login process using Docker:

## Step 1: Get the Docker Image

You can either pull the pre-built image from the GitHub Container Registry (GHCR) or build it locally.

### Option A: Pull the Published Image (Recommended)

```bash
docker compose pull
```

### Option B: Build the Image Locally

If you want to modify the source code or build it from scratch:

```bash
docker compose build
```

## Step 2: Run the Interactive Setup via Docker

We need to run `login_setup.py` interactively so you can input your username, password, and the MFA code when prompted. 

Run the following command. It overrides the default entrypoint to execute the setup script in an interactive terminal attached to the container:

```bash
docker compose run --rm -it --entrypoint "python login_setup.py" monarch-mcp
```

Follow the prompts on your screen:
1. Answer `y` if you have MFA enabled.
2. Enter your Monarch Email and Password.
3. When prompted, enter your Two-Factor (MFA) Code.

## Step 3: Verify the Session Token

Once login is successful, the script will output `✅ Session saved securely to keyring!`. 

Because we configured `PYTHON_KEYRING_BACKEND=keyrings.alt.file.PlaintextKeyring` in the `Dockerfile`, the token is actually saved to a plaintext configuration file within the container. Since the container maps `/app/data` to the `monarch-data` volume, your session token is preserved.

## Step 4: Run the MCP Server

Now that the session token is persistently stored in the Docker volume, you can run the server.

If testing locally:
```bash
docker compose up
```

If configuring for **Claude Desktop**, you'll need to specify the `docker run` command with standard stdio capabilities and the correct volume mount. Update your `claude_desktop_config.json`:

**For the published GHCR image:**
```json
{
  "mcpServers": {
    "monarch": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "--volume",
        "monarch-data:/app/data",
        "ghcr.io/robcerda/monarch-mcp-server-docker:latest"
      ]
    }
  }
}
```

**For the locally built image:**
```json
{
  "mcpServers": {
    "monarch": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "--volume",
        "monarch-data:/app/data",
        "monarch-mcp-server-docker-monarch-mcp"
      ]
    }
  }
}
```
*(Make sure to match the exact volume name and image name generated by your pull or build step).*
